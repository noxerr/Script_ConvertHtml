The table style defines the table features. You can define the following parameters in each table style: Name, Attributes and Apply to.

The table styles are defined by different Property Fields that can be added or deleted at any time.

New table style

To create a new table style, press the New style button in the Style Manager dialog box. Follow the steps of the table style wizard to define the new table style parameters:

  • Style name
  • Object types: more than one object type can be selected
  • Schedule table fields: specify the field name and which property it applies to.
  • Quantity field: option to specify the field in which to group the table elements

Edit a table style

Once the new table style is created it appears on the list in the left panel. Select the table style you wish to define and use the tabs that appear in the right panel to enter the table parameters:

Add a property field: New property fields can be added to a table style. Select the table style, right-click to display the context menu and then select New > Table property field.


Style Manager dialog box for the table object, "Apply to" tab.

General style editing
You can use the vaTableStyles command or, alternatively,
  • Use the vaStyleProperties command, then select a table and press Enter, or
  • Right click on , then select a table and press Enter, or
  • Select a table and open the context menu by pressing the right mouse button for a while then select Table > Style Properties, or
  • Press Alt, Shift or Ctrl then double click on a table object.
The changes applied to a style are applied to all elements with that style.
Object-specific editing
  • Use the vaProperties command, then select a table and press Enter, or
  • Click on , then select a table and press Enter, or
  • Select a table and open the context menu by pressing the right mouse button for a while then select Table > Properties, or,
  • Double click on a table object.
The changes are applied only to the style of the selected element.

Attributes

Attributes define the features of each table style for display and print purposes. The valid attributes are:

  • Layer, Visibility, Text style, Dimension style, Color, Linetype, Line print width, Line print color, Pattern, Pattern scale, Pattern angle and Pattern color.

Apply to

This tab specifies the type of objects displayed on the table. The command automatically filters the selection, so that only the type of objects selected are displayed.

The style manager dialog box of the Table object displays two more tabs when you select one of the columns of each table style: Group and Property.

The quantity and column type of each table style can be defined by the user, depending on what needs to be quantified. For the three default table styles, the predefined columns are:

  • Openings style: Quantity, Reference, Width, Height, Elevation and Opening Side.
  • Spaces style: Name, Perimeter and Area.
  • Walls style: Quantity, Style, Length, Area and Volume.
Group

This tab is used to define the common features of a quantified object so that it can be considered as part of the same group.
This tab only appears when you select the column on the table corresponding to the Quantity (used in the opening elevations).


Style Manager dialog box for the Table object, "Group" tab.

Property

This tab is used to define the property type of a quantified object that must be displayed in each table column.
This tab is displayed only when you select any of the table columns, except in the case of Quantity.

Sort

This tab is displayed when selecting a table style. It is used to set a criterion to order the different property fields within the table style and also to order the values of each property field alphabetically or numerically.

Click on the Add button to add a new property field which will establish the first criterion to order the values of that table style. Click on this property field name to select a different property field. Select the Order type (Ascending or Descending) besides the property field.

You can add more property fields that will provide new criteria to order the values in table styles. Use the arrows at the right-bottom corner of the dialog to change the property fields order.
