Plan View

Keyboard:
vaPlanView
Icon:
left click [right click]
Documentation toolbar:
Menu:
VisualARQ > Plan View > Plan View

The vaPlanView command creates an entity that represents the 3D objects projected on a horizontal plane, although it does not show the hidden faces.

The Plan View is created according to an existing level in the model, so a building and a level must be created in the document before running this command.

After running the vaPlanView command, the Plan View insert dialog box will appear, showing a list of plan view styles. This dialog is followed by a dialog which asks the level to display in the Plan View. After closing these dialogs, follow the steps from the command line. Press the right mouse button or ENTER to accept and go to the next step.


Insert dialog 1: Select the plan view style


Insert dialog 2: Select the plan view level

Steps:

  1. Select the Plan View style you wish to use.
  2. In the following dialog box, select the Level you wish to represent in 2D horizontal plan, according to the existing levels created in the document. (If no levels are created, exit the command and run the vaLevels command to create a new building and level)
  3. Define the boundary area of the objects to include in the Plan View. There are two options available:
    1. Rectangular boundary area: specify the two opposite corner points of the rectangular area.
    2. Custom boundary area: select the FromCurve option in the command line and pick a closed and planar curve (previously created) that will define the area of the objects displayed in the Plan View.
  4. Specify an insertion point. Left click anywhere on the screen to insert the plan view.

The command ends when the 2D drawing of the plan view is inserted in the specified insertion point. This drawing is created in the current layer.

Control Points

The Plan view drawing has the same control points as the boundary area selected to define the objects to include in the Plan View.

Plan View parameters

The following parameters can be defined by each Plan View object, after they have been created. They can be edited in the VisualARQ Properties section (in the Rhino Properties Panel ).
  • Style: the style defines the attributes of Plan Views.
  • Level: option to change the level displayed in the Plan View.
  • View depth: it is measured from the Cut Plane position of the selected level and indicated within parentheses in each option.
    • Level: the plan view will show the objects located at the selected level only.
    • Level below: the plan view will show the objects located beneath the selected level and the level below.
    • Unlimited: the plan view will show everything located beneath the selected level.
  • Title: the level name is taken by default as the title of the Plan View. It can be changed by any other text.
  • Scale: scale factor in which the Plan View is displayed.
  • Auto update: option to enable an automatic update of the Plan View when the Level properties (Name, Elevation or Cut Plane values) are modified.
  • Show boundary: option to show or hide the Plan View boundary area.
  • Show dimensions: option to draw the dimension lines lined up with the exterior walls, roofs and openings of a project when the Plan View is created.
  • Tolerance factor: this value determines the level of detail the plan view is generated with. The higher this value is, the less detailed the plan view will be.
